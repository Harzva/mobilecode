# MobileCode 代码冗余分析与优化计划

## 项目规模
- 181+ 个 Dart 文件
- 146,357+ 行代码
- 6 大层次：core / models / services / providers / screens / widgets

## 分析维度

### 1. 重复代码检测 (Duplicate Code)
- 跨文件的相同/相似代码块
- 重复的常量定义
- 重复的工具函数
- 重复的 UI 模式

### 2. 过度设计检测 (Over-Engineering)
- 不必要的抽象层
- 过度封装的类
- 只有一处使用的通用组件
- 空洞的接口/基类

### 3. 未使用/死代码检测 (Dead Code)
- 未使用的导入
- 未调用的函数/方法
- 未使用的变量
- 不可达代码

### 4. 结构冗余 (Structural Redundancy)
- 可以合并的相似文件
- 分裂过细的服务/Provider
- 重复的状态管理逻辑
- 重复的 API 调用模式

### 5. 主题/样式冗余 (Theme Redundancy)
- 硬编码的颜色/字体大小
- 重复的装饰定义
- 与主题系统冲突的自定义样式

## 并行分析分组

| 组 | 负责文件 | 分析重点 |
|----|----------|----------|
| A - 核心与模型 | core/*, models/* | 重复常量、过度抽象、主题冗余 |
| B - 服务层 | services/* (40+文件) | 重复API模式、过度封装、可合并服务 |
| C - Provider层 | providers/* (30+文件) | 重复状态逻辑、空洞Provider、可合并 |
| D - UI层(上) | screens/* (前20个) | 重复UI模式、硬编码样式、重复构建逻辑 |
| E - UI层(下) | screens/* (后20个) + widgets/* | 重复组件、重复动画、冗余构建 |

## 优化策略
1. 提取公共代码到共享工具
2. 合并高度相似的服务/Provider
3. 删除死代码和未使用导入
4. 统一硬编码值为主题引用
5. 简化过度设计的抽象
